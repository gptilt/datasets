3. **Cleaning:** Each Cargo table is deduplicated on its canonical key (`GameId`, `MatchId`, `OverviewPage`) and projected to a narrow, stable schema: `ScoreboardGames` → `games` (Team1 = blue side, Team2 = red side), `MatchSchedule` → `matches` (series), and `Tournaments` → `tournaments`. Champion picks/bans are split into lists of champion names (role-ordered for picks), and a `source_url` back to the originating wiki page is reconstructed. The clean layer is **overwrite-only** — each run reflects the *latest snapshot* of Leaguepedia, not historical state.
